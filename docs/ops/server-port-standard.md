# 服务器端口规范

本规范用于阿里云 ECS 上所有自部署应用。新增应用必须先登记端口，再配置 PM2 和 Nginx。

## 总原则

| 规则     | 要求                                                       |
| -------- | ---------------------------------------------------------- |
| 入口统一 | 公网只开放 Nginx 的 `80/443`                               |
| 应用绑定 | Web 应用只监听 `127.0.0.1`，禁止监听 `0.0.0.0`             |
| 端口登记 | 新端口先写入 `/etc/app-port-registry.yml` 和本仓库规范文档 |
| 命名统一 | PM2 名称使用 `{app}-{role}-{env}`，例如 `shuo-web-prod`    |
| 健康检查 | 部署后必须检查 `http://127.0.0.1:{port}`                   |
| 反代统一 | Nginx `proxy_pass` 必须指向登记端口                        |

## 端口段

| 端口段        | 用途                      | 示例                 |
| ------------- | ------------------------- | -------------------- |
| `16000-16999` | 运维、调试、观测工具      | OpenCode、debug 面板 |
| `17000-17999` | 内部 API、Worker、Webhook | 自动化任务 API       |
| `18000-18999` | 线上 Web 应用             | 个人站、内容站、官网 |
| `19000-19999` | 预发、测试、灰度环境      | staging 站点         |

`3000-3999` 属于框架默认端口区，禁止作为长期生产端口使用。

## 新应用申请流程

1. 选择端口段：线上 Web 选 `18xxx`，预发选 `19xxx`。
2. 检查服务器占用：`ss -ltnp | sort -k4`。
3. 更新 `/etc/app-port-registry.yml`。
4. 更新仓库中的 `docs/ops/server-port-standard.md`。
5. 应用启动命令绑定 `127.0.0.1:{port}`。
6. Nginx 配置 `proxy_pass http://127.0.0.1:{port}`。
7. 部署后执行健康检查。

## 推荐 PM2 启动格式

```bash
PORT=18301 HOSTNAME=127.0.0.1 pm2 start "corepack pnpm@8.15.9 start" --name shuo-web-prod
```

## 禁止事项

- 禁止直接使用 `3000`、`3001`、`5173`、`8000`、`8080` 作为长期生产端口。
- 禁止应用直接监听公网地址。
- 禁止未登记端口直接上线。
- 禁止 Nginx 配置和 PM2 实际端口不一致。
