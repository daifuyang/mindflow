# zerocmf 插件

> 基于 zerocmf 架构的企业级插件：CRM、商城、小工单。

## 基本信息

- **优先级**：P1
- **状态**：规划中
- **第一性原理**：企业不需要重复造轮子，一套系统解决常见业务需求
- **依赖**：zerocmf 核心架构稳定后开发

## 插件列表

### 1. CRM 插件

**定位**：客户关系管理，适合销售团队

| 功能 | 说明 |
|------|------|
| 客户管理 | 客户信息、跟进记录 |
| 销售漏斗 | 线索 → 成交全流程 |
| 数据报表 | 销售业绩可视化 |

### 2. 商城插件

**定位**：B2C 商城，适合有电商需求的企业

| 功能 | 说明 |
|------|------|
| 商品管理 | SKU、库存、价格 |
| 订单管理 | 订单流程、售后 |
| 支付集成 | 微信、支付宝 |
| 会员系统 | 积分、等级 |

### 3. 小工单插件

**定位**：轻量工单系统，适合客服/售后团队

| 功能 | 说明 |
|------|------|
| 工单创建 | 多渠道接入 |
| 工单流转 | 分配、转派、升级 |
| 满意度评价 | 客户反馈 |

## 技术架构

所有插件基于 zerocmf 插件运行时：

```
zerocmf/
├── src/plugins-runtime/     # 插件运行时
├── src/plugins/modules/     # 插件模块
│   ├── crm/                # CRM 插件
│   ├── shop/               # 商城插件
│   └── ticket/             # 小工单插件
└── prisma/schema/          # 数据模型
```

## 开发顺序

建议按照商业价值优先级：

1. **小工单**（P1）- 最简单，先跑通插件流程
2. **CRM**（P1）- 需求明确，变现能力强
3. **商城**（P2）- 实现复杂，放在最后
