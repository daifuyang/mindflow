# 自媒体任务规范

> 标准化内容生产流程，确保每篇内容质量稳定、可追踪、可复盘。

## 任务状态定义

| 状态 | 说明 | 下一步 |
|------|------|--------|
| `idea` | 选题想法，未确认 | 评估后转为 `planned` 或丢弃 |
| `planned` | 已确认排期 | 开始写作 |
| `drafting` | 正在起草 | 完成后进入 `review` |
| `review` | 待审核/修改 | 修改后进入 `ready` |
| `ready` | 待发布 | 发布后进入 `published` |
| `published` | 已发布 | 等待数据复盘 |
| `reviewed` | 已复盘 | 归档 |

## 任务字段规范

每篇文章的 frontmatter 应包含以下信息：

```yaml
title: 文章标题
date: YYYY-MM-DD（发布日期）
status: 当前状态
platform:
  - 掘金
  - 微信公众号
tags:
  - 标签1
  - 标签2
order: 排序（同分类内）
```

## 内容质量标准

### 标题要求
- 包含关键词，利于 SEO
- 有吸引力但不标题党
- 长度控制在 20-30 字

### 正文要求
- 开篇 3 句话内说明文章价值
- 代码示例必须可运行
- 配图清晰，有标注说明
- 文末引导关注/互动

### 发布检查清单
- [ ] 标题检查：关键词 + 吸引力
- [ ] 排版检查：段落间距、代码高亮
- [ ] 配图检查：清晰度、版权
- [ ] 链接检查：所有外部链接有效
- [ ] 引导语：文末有关注/互动引导
- [ ] 平台适配：掘金和公众号格式分别检查

## 复盘规范

### 每篇文章发布后 3 天记录数据
- 阅读量
- 点赞/收藏数
- 评论数
- 涨粉数

### 每月月底做月度复盘
- 各平台数据汇总
- 最佳表现文章分析
- 最差表现文章分析
- 下月调整方向
